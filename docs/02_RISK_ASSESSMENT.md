# Risk Assessment Matrix

Τα παρακάτω ρίσκα δεν μεταβάλλουν τα εγκεκριμένα milestones του έργου. Λειτουργούν ως τεχνική τεκμηρίωση των περιορισμών που εντοπίστηκαν κατά την αρχική διερεύνηση και ως βάση για mitigation μέσα στο υφιστάμενο χρονοδιάγραμμα.

| Εύρημα / Κίνδυνος (Risk) | Πιθανότητα (Likelihood) | Impact (Αντίκτυπος) | Mitigation (Στρατηγική & Μέτρα Μετριασμού) |
|---|---|---|---|
| Ετερογενής ποιότητα PDF αρχείων. | Υψηλή | Υψηλό | Ταξινόμηση του πιλοτικού δείγματος ανά χρονολογία, τύπο PDF και ποιότητα εξαγωγής. Το pipeline δεν θεωρεί όλα τα PDFs ισοδύναμα. |
| Παλαιά έγγραφα, ειδικά πριν το 1990, μπορεί να έχουν κακή σάρωση, δακτυλογραφημένο κείμενο ή δύσκολο OCR. | Υψηλή | Υψηλό | Τα παλαιά έγγραφα αντιμετωπίζονται ως κατηγορία αυξημένου ρίσκου. Παραμένουν αναζητήσιμα μέσω metadata και οδηγούνται σε OCR/manual review workflow όπου απαιτείται. |
| Το υπάρχον text layer σε κάποια PDFs μπορεί να είναι ελλιπές, θορυβώδες ή λανθασμένο. | Μεσαία προς Υψηλή | Υψηλό | Εφαρμογή quality checks πριν τη δημιουργία embeddings. Το εξαγόμενο κείμενο χαρακτηρίζεται ως κατάλληλο για indexing μόνο όταν περνά βασικά κριτήρια αναγνωσιμότητας. |
| OCR output μπορεί να περιέχει λάθη, αλλοιώσεις ελληνικών χαρακτήρων ή θόρυβο. | Υψηλή | Υψηλό | Δοκιμή OCR λύσεων στο πιλοτικό δείγμα, με βασικό υποψήφιο το Surya OCR και εναλλακτικά EasyOCR/docTR. Το OCR χρησιμοποιείται ως fallback και όχι άκριτα ως μοναδική πηγή αλήθειας. |
| Κίνδυνος δημιουργίας embeddings από κακό ή αλλοιωμένο κείμενο. | Υψηλή | Πολύ Υψηλό | Κάθε σελίδα ή chunk χαρακτηρίζεται ως `index_content`, `metadata_only`, `skip` ή `manual_review`. Μόνο το αξιόπιστο περιεχόμενο μπαίνει στη vector database. |
| Σελίδες εξωφύλλων, περιεχομένων, πινάκων ή χαρτών μπορεί να αξιολογηθούν λανθασμένα ως αποτυχημένη εξαγωγή. | Μεσαία | Μεσαίο | Χρήση ξεχωριστής κατηγορίας `metadata_only`, ώστε τέτοιες σελίδες να μη χρησιμοποιούνται ως κύριο επιστημονικό κείμενο αλλά να μην απορρίπτονται πλήρως. |
| Καθαρεύουσα, παλαιότερη ορολογία και πολυτονικά στοιχεία μπορεί να μειώσουν την ποιότητα αναζήτησης. | Μεσαία προς Υψηλή | Μεσαίο προς Υψηλό | Διατήρηση `original_text` και δημιουργία `normalized_text` για search. Αξιοποίηση controlled vocabulary ή mapping παλαιών και σύγχρονων όρων όπου είναι διαθέσιμο. |
| Το LLM μπορεί να παράγει πειστικές αλλά μη τεκμηριωμένες απαντήσεις. | Μεσαία | Υψηλό | Ο AI Assistant απαντά μόνο με retrieval από τεκμηριωμένες πηγές. Κάθε απάντηση συνοδεύεται από αναφορά σε έγγραφο και σελίδα. Αν δεν υπάρχουν επαρκή στοιχεία, το δηλώνει ρητά. |
| Vision LLM ή local LLM μπορεί να “διαβάσει” λανθασμένα δύσκολες σελίδες και να παράγει hallucinated κείμενο. | Μεσαία | Υψηλό | Χρήση vision/local LLMs μόνο ως βοηθητικό fallback για manual review ή περιγραφή εικόνων/χαρτών, όχι ως authoritative OCR source για αυτόματο scientific indexing. |
| Η πιλοτική φάση των 100 εκθέσεων μπορεί να μην είναι αντιπροσωπευτική του συνολικού αρχείου. | Μεσαία | Μεσαίο προς Υψηλό | Επιλογή δείγματος με διαφορετικές χρονολογίες, θεματικές περιοχές, τύπους εγγράφων και ποιότητες PDF. Τεκμηρίωση της σύνθεσης του δείγματος. |
| Η μετάβαση από πιλοτικό δείγμα στο σύνολο των εκθέσεων μπορεί να έχει απρόβλεπτες δυσκολίες. | Μεσαία | Υψηλό | Staged rollout. Πρώτα επεξεργασία καθαρών/νεότερων εγγράφων και μετά σταδιακή ένταξη πιο δύσκολων κατηγοριών με ειδικά workflows. |
| Υψηλό κόστος χρήσης paid LLM / embedding models σε πλήρη κλίμακα. | Μεσαία | Μεσαίο προς Υψηλό | Περιορισμός χρήσης paid models σε αξιόπιστο και χρήσιμο περιεχόμενο. Caching embeddings και απαντήσεων όπου είναι εφικτό. Εξέταση local/open-source επιλογών για OCR και βοηθητικές εργασίες. |
| Ελλιπή ή ασυνεπή metadata από KOHA μπορεί να επηρεάσουν classification και retrieval. | Μεσαία | Μεσαίο | Αρχικός έλεγχος ποιότητας metadata. Όπου λείπουν κρίσιμα πεδία, όπως χρονολογία, χρήση fallback από τίτλο, PDF metadata ή manual tagging στο πιλοτικό δείγμα. |
| Το hybrid retrieval μπορεί να επιστρέφει πολλά ή ασύνδετα αποτελέσματα. | Μεσαία | Μεσαίο | Reranking που συνδυάζει metadata relevance, keyword match, vector similarity και ποιότητα extraction. Αξιολόγηση με πραγματικά ιστορικά ερωτήματα. |
| Οι χρήστες μπορεί να εμπιστεύονται υπερβολικά τις AI απαντήσεις. | Μεσαία | Υψηλό | Το interface εμφανίζει καθαρά πηγές, σελίδες και επίπεδο βεβαιότητας. Δεν εμφανίζεται απάντηση χωρίς τεκμηρίωση. |
| Χαμηλή αποδοχή από χρήστες αν τα πρώτα αποτελέσματα δεν είναι καλύτερα από την υπάρχουσα αναζήτηση. | Μεσαία | Μεσαίο προς Υψηλό | Ορισμός baseline με ιστορικά ερωτήματα και αξιολόγηση ανά use case. Παρουσίαση του PoC ως συμπληρωματικό εργαλείο και όχι ως αντικατάσταση της υπάρχουσας αναζήτησης. |
| Τεχνική πολυπλοκότητα διασύνδεσης με KOHA / υπάρχον περιβάλλον βιβλιοθήκης. | Μεσαία | Μεσαίο | Πρώιμη τεχνική διερεύνηση διαθέσιμων exports, APIs και schema. Το PoC ξεκινά με σταθερό export αν η άμεση διασύνδεση καθυστερεί. |
| Έλλειψη audit trail για extraction, embeddings και απαντήσεις. | Μεσαία | Υψηλό | Αποθήκευση extraction method, quality status, embedding model/version, query logs και answer logs. Κάθε απάντηση είναι αναπαραγώγιμη και ελέγξιμη. |
| Προβληματικά έγγραφα μπορεί να καθυστερήσουν το PoC. | Υψηλή | Μεσαίο | Το PoC δεν απαιτεί επιτυχία σε όλα τα PDFs. Επιτυχία θεωρείται λειτουργικό pipeline με τεκμηριωμένη ταξινόμηση επιτυχών, αποτυχημένων και manual-review περιπτώσεων. |
| Κίνδυνος scope creep λόγω προσπάθειας επίλυσης όλων των OCR προβλημάτων από την αρχή. | Υψηλή | Υψηλό | Η πιλοτική φάση περιορίζεται σε τεκμηρίωση ρίσκου και αξιόπιστη επεξεργασία αντιπροσωπευτικού δείγματος. Τα δύσκολα ιστορικά έγγραφα μπαίνουν σε ξεχωριστό backlog/fallback workflow. |

## Κύρια τεχνική επισήμανση

Ο βασικός τεχνικός κίνδυνος του έργου δεν είναι η χρήση LLM ή vector database, αλλά η αξιοπιστία της εξαγωγής κειμένου από ανομοιογενή και παλαιά PDFs.
